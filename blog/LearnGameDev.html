How I Plan to Learn Game Dev

Some History

The main reason I got into programming, as a child, was actually game development. I'd always been intrigued by stories; whether in books, movies, or theatre. I also enjoyed the action and control video games granted, a lot. When asked where I saw myself ten years in the future, I naturally thought 'Video Game Developer'.

The path to becoming one was complex, though. When I tried figuring out a simple roadmap to becoming a game developer, I was bombarded with a lot of contradictory advice. From 'You need to know programming!' to 'None necessary'. 'Just use a game engine!' to 'Create your own'.

Ultimately, I decided to create my own engine, since that would serve as a good test: if I could create a game engine, alone, then I could probably design decent games. If I couldn't, I didn't deserve to. Besides, I figured creating a game engine would teach you a lot of the skills required in game development too.

But the contradictions didn't end there. If you wanted to create an engine, you had to choose between C++ and C# too. Problem was: either language was far too advanced for my liking. Nobody recommended I learn C++ or C# as a first language, though, so I was stuck at step one. I had to learn programming, probably in its full, before becoming a true game developer. The reality stuck in. Becoming a game developer wasn't going to be easy. But, with childish naivety, I stashed my doubts away and started learning... C.

I was young back then, maybe 10, so I find it easy to forgive myself, but I wasted an year laboring away at C, of all languages, and developed an annoyance at programming. So, I gave up. I'd figured that C was a good start to learning C**++** since 'C++ was just a new version of C'.

A few years later, I finally discovered that Python or Lisp would've been better starting languages, and I began work on them. Unfortunately, with school and whatever else, I didn't really get far before forgetting about the whole thing.

Now, I'm finally done learning Python in-depth, and I'm ready to take a crack at C++, and game development as a whole. I'm going to stick true to my original plan of developing my own game engine, because it seems like a sound idea, but it's just a guide for becoming a game developer. I also prefer reading, and most of my plan will be books, but I'll try and include courses.

Much of this 'plan' is probably redundant. I just went over what would cover the basics and advanced of game development and all the other dependents. I haven't (obviously) gone through it yet. Once I do, I'll probably fix it up and write up on it with more detail.

Bold mean mandatory.

The Plan

The first step of the plan is to learn programming in-depth. Game development is just a branch-off of general development. You can follow OSSU or my own plan. Learning development in-depth is not a requirement, but it provides a good backup, I imagine.

Learning C++

With that out of the way comes the mammoth task of learning C++. I divided this into three parts: Beginning, Practice, and STL (Standard Library). Asides from 'beginning', I plan to study the rest simultaneously with the next topics.

  • Beginning: C++ Primer
  • Practice:
    • Effective C++,
    • Effective Modern C++,
    • Optimised C++
  • STL: The C++ Standard Library

The Maths & Physics

I was planning on skipping this part, and the other topics on 3D stuff generally, since I was never planning on making a 3D game engine, but it might be useful in 2D game dev as well. The mathematics of programming is a separate topic, and for that you can refer to my guide.

  • Maths:
    • Foundations of Game Engine Development: Volume 1 ,
    • Mathematics for 3D Game Programming and Computer Graphics,
    • Computational Geometry: Algorithms and Applications
